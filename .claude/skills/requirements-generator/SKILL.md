# requirements-generator

特定の機能や環境の要件定義ファイル（`.md`）を `.docs/requirements/` 配下に生成するスキル。

## Usage

```
/requirements-generator <対象の説明>
```

例:
- `/requirements-generator 会員管理機能`
- `/requirements-generator 予約システム`
- `/requirements-generator 本番環境のインフラ構成`

## Behavior

1. ユーザーから対象の説明を受け取る
2. 既存の要件定義ファイルを `.docs/requirements/` から読み取り、重複や整合性を確認する
3. 対話的にヒアリングを行い、要件を明確化する
4. 以下のテンプレートに従って要件定義ファイルを生成する
5. ファイルは `.docs/requirements/` 配下に `<slug>.md` として保存する

## Template

生成するファイルは以下の構成に従うこと:

```markdown
# <機能名 or 環境名>

## 概要
- 目的・背景を簡潔に記述

## スコープ
- 対象範囲（何を含み、何を含まないか）

## 機能要件
- [ ] 要件1
- [ ] 要件2
- ...

## 非機能要件
- パフォーマンス
- セキュリティ
- 可用性
- スケーラビリティ
（該当するもののみ記載）

## データモデル
- 主要なエンティティとその関連（必要な場合）

## API仕様（該当する場合）
- エンドポイント一覧
- リクエスト/レスポンス概要

## 依存関係
- 他の機能やサービスとの依存

## 制約・前提条件
- 技術的制約、ビジネス上の前提

## 未決事項
- [ ] 検討が必要な項目
```

## Rules

- ファイル名は英語のケバブケース（例: `member-management.md`）
- 内容は日本語で記述する
- 既存の要件定義との整合性を保つ
- 不明な点はユーザーに確認してから記載する（推測で埋めない）
- テンプレートのセクションは対象に応じて取捨選択してよい
- 生成後、ユーザーにレビューを依頼する
